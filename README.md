## 製品概要
リサイクル×Teck：捨てられてしまうごみを<b>ゴブジェクト</b>として生まれ変わらせる、AI工作提案アプリです。</br>
家庭や学校などで不要になったものを撮影すると、AIがその素材や形状を解析し、リサイクル工作のアイデアと作り方を提案してくれます。</br>
親子での工作や学生の自由研究、地域のエコイベントなど、幅広い場面で活用できることを想定しています。</br>
「再利用したい」という想いを「すぐに作れる」行動へとつなげ、創造的で楽しいリサイクル体験を提供します。</br>
捨てるはずだったものが、自分だけの宝物に生まれ変わる——そんな体験を、このアプリが実現します。

### 背景(製品開発のきっかけ、課題等）
近年、SDGsの浸透により「再利用」「リサイクル」への関心は高まっていますが、実際には「何を作ればいいのか分からない」「考えるのが面倒で結局捨ててしまう」といった課題が多く存在します。</br>
特に家庭や学校では、工作や自由研究のネタを探す負担が大きく、せっかくの再利用意欲が行動に結びつかないケースも少なくありません。</br>
そこで、AIの力で考える手間をなくし、誰もが簡単にリサイクルを楽しめる仕組みを作るために<b>「ゴブジェクター」</b>を企画しました。</br>

### 製品説明（具体的な製品の説明）
「ゴブジェクター」は、ユーザーが不要な物をカメラで撮影すると、AIが自動で素材や形状を分析し、その特性に合わせた工作アイデアを難易度別に提示します。</br>
提案には、作り方の手順を説明する文章と図解、そして完成イメージも含まれています。</br>

### 特徴
#### 1. 簡単なステップで自動提案
ユーザーは、不要になった物を撮影するだけで、AIが素材や形状を自動で解析し、それに適したリサイクル工作アイデアを瞬時に提案します。</br>
思考の手間を減らし、誰でも気軽に「再利用」を始められる体験を提供します。</br>
#### 2. 難易度別・図解付きのわかりやすいガイド
提案された工作アイデアは、難易度順に分かれており、年齢やスキルに合わせて選択可能です。</br>
各アイデアには、手順の説明文・作業図・完成イメージが付属しており、子どもから大人まで安心して取り組むことができます。</br>
#### 3. 学びと創造で楽しいリサイクル体験
家庭や学校、地域イベントなど、さまざまな場面で活用でき、親子のコミュニケーション促進やSDGs教育の実践にもつながります。</br>
工作を通じて「作る楽しさ」と「環境への気づき」を同時に体験できる、次世代のリサイクル支援アプリです。</br>

### 新規性
本プロジェクトは、複数の最先端AIモデルを組み合わせ、ユーザーの創造的なアップサイクル活動を支援するというユニークな体験を提供します。

1.  **マルチモーダルAIによる創造支援**</br>
    ユーザーが提供する廃材の**画像**と**テキスト情報**をインプットとし、Gemini APIによる**構造化されたDIYレシピの生成**と、Tripo AIによる**完成イメージの3Dモデル化**を同時に行います。専門性の異なる複数のAIサービスを連携させ、「アイデア創出」から「具体的な手順の理解」、「完成形の視覚化」まで、サポートするパイプラインを構築している点が最大の特徴です。

2.  **教育・SDGsへの応用可能性**</br>
    単なる情報提供ツールではなく、ユーザーの手元にある「不要なもの」から新しい価値を生み出す創造的なプロセスを具体的に支援します。DIYのレシピは小学生にも理解できるような簡単な言葉で生成され、夏休みの自由研究などに活用できる可能性を秘めています。

3.  **堅牢なバックエンド設計**</br>
    AIからの出力を安定させるため、詳細な役割設定や出力形式の厳密な指定を組み込んだ**高度なプロンプトエンジニアリング**を実装しています。また、APIからの応答が期待した形式でない場合にAI自身に修正を促す**自己修復メカニズム**や、複数の外部API呼び出しを並列処理して待ち時間を短縮する非同期処理など、システムの堅牢性とUXを向上させるための工夫を凝らしています。

4.  **インタラクティブなUI/UX**</br>
    生成された複数のDIYアイデアをカード形式で比較検討でき、選択したアイデアは詳細な手順とインタラクティブな3Dモデルで確認できます。これにより、ユーザーは直感的に完成イメージを掴みながら、楽しく制作に取り組むことができます。

### 今後の展望
今後は、ユーザーが作成した作品やアイデアを投稿・共有できる仕組みを整え、工作アイデアが循環するコミュニティを構築します。</br>
また、AIモデルの改良を進め、複数の素材が組み合わさった複雑な形状にも対応できるよう進化させていきます。</br>
さらに、解析や生成の速度向上にも取り組み、ユーザーがよりスムーズにアイデア提案を受け取れる体験を目指します。</br>

### 注力したこと（こだわり等）
本開発では、AIの解析・生成の「精度」「速度」「安全性」を重視しました。</br>
ReactとFastAPIを分離し、非同期通信で画像解析から提案生成までを高速化。</br>
Cloud Vision APIで得た素材情報をもとに、Gemini APIへ最適化されたプロンプトを自動生成し、撮影画像から短時間で多様な提案を返せるようにしました。</br>
また、生成結果を難易度別に整理し、図解や手順付きで分かりやすく提示する仕組みを導入。幼い子供でも使いやすいポップなUIを意識しながら作成しました。</br>
FastAPIでは環境変数管理や.gitignore設定を徹底し、APIキー流出防止などのセキュリティにも配慮しました。</br>
さらに、Tripo AIによる3D再構成を組み合わせ、提案内容を視覚的に理解できる体験を実現しました。

### 独自技術
本プロジェクトの中核をなす、AI連携とアイデア生成に関する独自開発技術です。

*   **マルチモーダルAIパイプラインの構築**
    *   **内容:** ユーザーがアップロードした廃材の画像とテキスト情報を基に、①Gemini APIによるDIYレシピ生成、②Tripo APIによる3Dモデル生成、という2つの異なるAI処理を並行して実行するパイプラインを構築しました。これにより、単一の入力からアウトプット（手順・材料・完成イメージ）を一度に提供できます。
    *   **実装:** `backend/app/main.py` にて `asyncio.gather` を用いて、GeminiとTripoへのAPIリクエストを非同期で並行処理し、ユーザーの待ち時間を短縮しています。

*   **堅牢なプロンプトエンジニアリングと自己修復システム**
    *   **内容:** 大規模言語モデルから安定して高品質なJSON形式のデータを取得するため、詳細な役割設定、厳密な出力形式の指定、そして具体的な回答例を盛り込んだプロンプトを自動生成するシステムを開発しました。さらに、APIの応答がJSONとして不正な形式であった場合に、AI自身にその修正を依頼する自己修復メカニズムを実装し、システムの安定性を高めています。
    *   **実装:** `backend/app/services/gemini.py` の `_format_prompt` 関数でプロンプトを構築し、`_repair_json_with_gemini` 関数でJSONの自己修復を行っています。

*   **インタラクティブな3Dモデルビューワー連携**
    *   **内容:** バックエンドから返されたDIYアイデアごとに、対応する3Dモデル（`.glb`形式）をフロントエンドで動的に表示します。ユーザーは生成された完成イメージを、Webブラウザ上で自由に回転・拡大しながら確認できます。
    *   **実装:** `frontend/src/pages/AIGeneration/index.tsx` にて、Googleが開発したWeb Componentである `<model-viewer>` をReactコンポーネントに組み込み、バックエンドからのレスポンスに応じて表示するモデルを切り替えるロジックを実装しました。
